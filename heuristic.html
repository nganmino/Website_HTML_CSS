Heuristic Expert Evaluation

Heuristic evaluation is a process of examining the usability of the user interface design, to detect the errors or limitations, to improve or adjust them according to the usability principles. In the 1990s, Jakob Nielsen and Rolf Molich generated user interface design guidelines, including:

Heuristic UX Evaluation graphic

Explaination

  • Visibility of system status: users should receive enough time and sufficient information of the system status, which is displayed in an easy format to understand.
  • Match between system and the real world: users should find the language and the concepts easy to understand without much prior knowledge of the system operations.
  • User control and freedom: users should find space for them to freely choose whether to go forward, or to stay in their current step, or to go backward, or undo their current actions on the system.
  • Consistency and standards: users should find the graphic elements and terminology logical and synchronized across the similar platform of the systems they are using.
  • Error prevention: users should be clearly guided and warned before committing an actions to prevent possible errors when exploring the interface.
  • Recognition rather than recall: users should not be challenged with their memories, but instead, be suggested and guided through their steps when using the interface.
  • Flexibility and efficiency of use: users should be allowed to tailor their own frequent actions on the same interface
  • Aesthetic and minimalist design: users should encounter the information that is relevant to their current action on the interface.
  • Help users recognize, diagnose and recover from error: users should be expected as being unable to understand the system language and concepts, so that the error message should be written in a plain language.
  • Help and documentation: users may ask for documentation to help them through the issues they have on the interface.
